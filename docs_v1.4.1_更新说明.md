# V1.4.1 更新说明

在 V1.4 基础上修复两个问题，不改变既有功能与配置。

## 1. 番号识别更准确（修复误判）

**问题**：刮削时仅靠文件名用「字母+数字」正则猜番号，遇到带站点/数字前缀的文件名会误判。例如
`hhd800.com@390JAC-234.mp4` 的真实番号是 `390JAC-234`，却被识别成 `HHD-800`，导致搜索元数据失败。

**修复**：

- **推送即标记（主修复）**：在前端搜索结果里点「推送下载」时，那条影片的番号是准确的。推送时会把
  「番号 ↔ 该磁力/种子特征」记录到 `config/push_hints.json`；下载完成刮削时优先用文件名（及其上级
  目录名）反查这份标记，命中则直接采用准确番号，未命中再回退文件名解析。
- **文件名正则增强（兜底）**：解析前先剔除 `hhd800.com@`、`[javbus.com]`、`www.xxx.cc-` 等站点/广告
  前缀；并新增对 `390JAC-234`、`259LUXU-1234` 这类「数字前缀番号」的支持。

> 对手动放入（未经推送）的文件，靠增强后的文件名解析；经本程序推送下载的文件，番号识别最准。

## 2. 服务地址填 `http://localhost` 时种子下载/外网访问修复

**问题**：相关服务地址写成 `http://localhost` 后，连通检测、推送、磁力链均正常，但 Jackett 返回的
`.torrent` 直链带着它自身地址（`http://localhost:9117/...`），外网点击打不开，推送种子（无磁力时）也会失败。

**修复**：种子直链统一改为「由后端代取、再转交」——

- 前端「种子」按钮指向后端新接口 `/api/jackett/download`，后端取回 `.torrent` 后以附件回传浏览器
  （文件名自动用番号），任意网络都能下载。
- 推送 qBittorrent 时：磁力链照旧直接交给 qB；`.torrent` 直链由后端先取回字节再以文件上传给 qB，
  qB 端不再碰到 `localhost`。
- 迅雷链接在仅有种子时也改用后端代理的绝对地址。
