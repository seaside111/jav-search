# V1.4.4-beta 更新说明

在 V1.4.3 基础上**专项优化 FC2**，让「最新片源」真正拿到市面最新、且详情更快更省。其它功能与配置不变。

## 1. FC2 最新片源接入 sukebei（解决「不够新」）

**问题**：fc2ppvdb 的「新着列表」会封顶在某个号（实测只到 ~489 万），够不到市面最新；且它首页不支持翻页、忽略 `per_page`，最新页还把「最新 + 人気老片」混排。

**优化**：

- FC2 最新改为**优先用 sukebei 发现番号**（种子站按 id 倒序＝最新、**直连不过盾、最快**），能拿到 fc2ppvdb 够不到的最新号；sukebei 够量时直接跳过较慢的 fc2ppvdb 首页。
- 抓回后**统一按 FC2-PPV 编号降序**取最新 N 条（编号越大越新），老热门片自动沉底。
- sukebei 挂了 / 关闭时，自动回退 fc2ppvdb 首页兜底。
- 新增 `backend/scrapers/_sukebei.py`。

## 2. 后台预抓 MissAV（标题/封面/样品图，低负担）

sukebei 卡片初始只有种子标题 + 封面。新增**后台串行 + 节流 + 直连 MissAV**（不过盾）慢慢抓最新片的**真标题 / 封面 / 样品图**灌入缓存：刷新后列表卡标题/封面变干净、点开详情**样品图秒出**。默认开、负担极低，可在设置里关闭或调条数。

## 3. FC2 详情改为 MissAV-only（点开即出、不再过盾）

FC2 无女优/标签需求，而 fc2ppvdb 在详情里唯一独有的就是这些、对最新片还常为空，却要花一次慢速过盾。**详情移除 fc2ppvdb，只用 MissAV**（标题/封面/样品图，命中预抓缓存则秒出），**彻底不碰 FlareSolverr**，也不再与 JavDB/刮削抢实例。fc2ppvdb 仍保留用于**关键词/女优名搜索**与**首页最新兜底**。

## 4. 新增设置（FC2 数据源分栏）

- 最新优先用 sukebei（默认开）
- 后台预抓样品图（默认开）+ 预抓条数（默认 20）
- fc2ppvdb 抓取页数（兜底用，默认 1）

## 涉及文件

- 新增 `backend/scrapers/_sukebei.py`
- `backend/scrapers/fc2.py`、`backend/config_manager.py`、`backend/main.py`、`frontend/index.html`
