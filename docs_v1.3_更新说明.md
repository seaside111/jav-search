# JAV Search V1.3 更新说明

本版本以 V1 为基础，针对**抓取数量上限**、**抓取速度**、**数据源扩展**、**首页空白**四个方面做了优化。

---

## 一、抓取数量上限突破 + 速度优化（核心重构）

### 问题
V1 的搜索流程是：**抓列表页 → 把每一条结果的「详情页」全部并发抓取**。
演员、标签、导演、简介这些信息只有详情页才有，所以抓 60 条就要发 60 个详情请求。
这导致数量上限被卡在 60，再多就会非常慢（每条都要单独请求一次详情页）。

### 解决方案：列表抓取 / 详情抓取「分离」
关键洞察——**列表页本身已经包含足够渲染卡片的信息**（番号、标题、封面、日期，JavDB 还带评分），
详情信息只在用户「打开详情 / 翻译」时才真正需要。因此把两步拆开：

| 阶段 | 做什么 | 速度 |
|------|--------|------|
| **列表抓取** `/api/search` | 只翻列表页，解析卡片级条目 | 极快：每个请求拿 ~30 条，抓 500 条只需十几个请求 |
| **详情抓取** `/api/details` | 按需补全演员/标签/导演/简介 | 每次只抓「当前页 + 下一页」约 24 条，而非 500 条 |

这正好实现了你描述的流程：
1. **先拿到总数和分页** → 列表抓取一次性返回全部条目（最高 500），前端即时分页。
2. **预抓前面两三页内容** → 前端渲染某页后，后台立即预取**下一页**的详情。
3. **点击下一页时再启动后续抓取** → 翻到新页时补全该页详情；若已预取则瞬间显示。

### 上限
- 单源抓取上限默认 **300**，可在「设置 → 抓取与展示」调到 **500**（硬上限 500，防止拖垮服务）。
- 多来源会按番号**合并去重**，因此实际展示数可能少于「上限 × 源数」。

### 前端行为
- 搜索后卡片**立即出现**（封面/番号/标题/日期），不再等所有详情。
- 详情到达后**就地升级**卡片正文（演员/标签等），且**不会覆盖**你已展开的资源抽屉或已翻译的内容。
- 点开「详情弹窗」时，若该条详情尚未加载，会先显示「正在加载详情…」再补全。

## 二、数据源扩展

### 评估结论
同类元数据站中，**与 JavBus 同源模板**的站点改造成本最低、最稳：

| 站点 | 类型 | 可操作性 | 说明 |
|------|------|----------|------|
| **AVSOX** | 无码 | ⭐⭐⭐⭐⭐ | 与 JavBus 同模板，复用解析器即可；本版已接入（沙箱实测可直连） |
| **AVMOO** | 有码/旧片 | ⭐⭐⭐⭐ | 同模板，收录大量较老番号；域名常变，已接入并支持自定义域名 |
| JavLibrary | 有码+评分 | ⭐⭐ | 结构不同、Cloudflare 较严，性价比低，本版**暂未接入**（后续可选） |
| DMM/FANZA | 官方 | ⭐⭐ | 需地区/年龄校验，反爬强，不适合直连抓取 |

### 本版新增
- **AVSOX**（无码专站）、**AVMOO**（有码/旧片）已加入，默认**关闭**，在「设置 → 数据源」勾选启用。
- 这两个站点共用 JavBus 的解析逻辑（见 `scrapers/_javbus_base.py`），维护成本低。
- 域名易变，可用环境变量覆盖（docker-compose 中已留注释）：
  - `AVSOX_BASE`（默认 `https://avsox.click`）
  - `AVMOO_BASE`（默认 `https://avmoo.website`）

> 注：AVSOX/AVMOO/JavDB 多数需要科学上网（代理）才能访问，与 JavBus 一致。

## 三、首页最新片源

未搜索时首页不再空白：自动调用 `/api/latest` 抓取所选数据源的**首页最新片源**并以卡片展示。
- 可在「设置 → 抓取与展示 → 首页展示最新片源」开关。
- 取自「设置中已勾选的数据源」，每源默认 24 条。
- 最新片源同样是列表级条目，点开卡片会按需加载详情，逻辑与搜索结果一致。

## 五、修订（Bug 修复）

### 1. JavBus 封面/详情图片不显示
- **原因**：图片代理 `/api/img` 原先只用单一固定 `Referer: javbus.com` 且只走代理；当 JavBus 走镜像/CDN、或返回防盗链验证页（HTTP 200 但内容是 HTML）时，会把验证页当成图片缓存，导致显示空白。
- **修复**：
  - **多 Referer 兜底**：依次尝试「站点 Referer → 图片自身 origin → 无 Referer」，兼容镜像/CDN。
  - **代理/直连兜底**：先用配置代理，失败再直连（部分图床直连可达）。
  - **内容类型校验**：只接受真正的图片响应（`image/*`），HTML 验证页不再被误存。

### 2. 新增数据源勾选保存后又恢复未勾选
- **原因**：浏览器可能缓存了 `GET /api/config` 的旧响应，重开设置时读回保存前的旧值。
- **修复**：前端读取配置加 `cache:'no-store'`，后端 `/api/config` 响应加 `Cache-Control: no-store`，确保保存后立即读到最新设置。

> 以上两项均已在本地浏览器实测验证：JavBus 封面正常渲染、AVSOX/AVMOO 勾选保存后重开仍保持勾选。

### 3. AVSOX 列表项「一直显示详情加载中」
- **原因**：AVSOX 详情页本身较慢（每条约 3-4s），一页 12 条补全需约 10-16s；卡片上的「详情加载中…」字样在这段时间一直显示，让人误以为卡住（JavBus 详情快所以不明显）。
- **修复**：
  - **去掉「详情加载中…」字样**：卡片用列表级信息（封面/番号/标题/日期）即已完整可看，演员/标签/出品等详情加载完成后**渐进式补入**，不再显示加载提示。
  - **详情补全增加单条超时（9s）+ 前端 30s 兜底中断**：个别慢/挂起的详情不会再拖垮整页或无限等待。
  - 并发数调优为 6，兼顾速度与稳定。

### 4. 几乎所有片子都没有「简介」
- **这不是抓取错误，而是数据源本身没有简介**：JavBus / AVSOX / AVMOO 的详情页**不提供剧情简介**字段，只有番号、标题、演员、标签、出品商等。只有 **JavDB** 偶尔带简介。
- 因此简介为空属正常现象；如需简介，可启用 JavDB 数据源（需代理/Cloudflare 通过）。详情弹窗中无简介时会显示「该来源未提供简介」。

### 5. 首页最新片源数量可按来源调节
- 首页最新抓取数量改为**按来源单独配置**（`latest_limits`）：
  - **JavBus 默认 100**（你要求提升到 100 左右），AVSOX/AVMOO/JavDB 默认 40。
  - 可在「设置 → 抓取与展示」分别调节 JavBus / AVSOX 的首页最新数量。
  - 各来源最新会按番号合并去重后展示（JavBus 在前、其它在后）。

> 以上均已实测：首页 JavBus 100 + AVSOX 24 = 124 条；翻到 AVSOX 页后卡片演员/标签在约 10s 内渐进补全，无「加载中」残留。

## 六、兼容性
- 配置项向后兼容：老的 `settings.json` 缺失新字段时自动取默认值（`max_results` 会沿用旧值，可在设置中调高）。
- API：`/api/search` 返回结构不变（仍是 `{success,mode,total,results}`），仅 `results` 内每项多了 `detail_loaded`、`sources` 字段；翻译、Jackett、图片代理、登录等接口完全不变。
